CI/CDによる自動化

Arch Linux時代

  • 適当なアプデ
    →AURビルド不備、破壊的変更による起動不可
  • nix=Infrastructure as Code (IaC)
    • CI/CDで予防できる！

flake.lockの自動更新

  • DeterminateSystems/update-flake-lock
    • flake.lockを更新→PR作成
  • 実際は依存先から順に更新
    • nvimdots→flakes→nixos-desktop-config

Deprecated optionの通知

  • nixos-unstableは仕様変更が多々ある
  • 式を評価
    →Evaluation WarningがあればISSUEを開く

ビルドテスト

  • システムビルドに必要な容量：60GB～
  • 標準Runnerは容量が足りない→空き29GB
  • easimon/maximize-build-space
    • 不要パッケージの削除, ディスク(LVM)割り当ての変更
    • /nix：96GB確保
  • ビルド時間：40分～

パブリックでのシークレット管理

  • パブリックリポジトリだと標準Runnerが無料！
  • 個人マシンには秘匿情報がいっぱい！
    • パスワード、ssh、APIキー…

Mic92/sops-nix

  • age, pgpでファイル暗号化
  • アクティベーション時に符号化
  • /nix/storeには暗号化ファイルが配置

Branch・Versioning戦略

そのNixOS
常にロールバックできる?

No

  • 既デプロイへのロールバック○
  • コードレベルは？
    • force push/pull …
  • NixOSにおいてコード=マシン状態 (IaC)
  • 適切な管理下でロールバックが初めて担保

ブランチ戦略：GitHub flow

  • 細かい変更を受容
  • mainは常に起動可能な状態を維持
    • →エラー無く動作する？

バージョン戦略

  • 起動可≠各Appがエラー無く動作
  • nixpkgs/modulesが
    アップストリームの仕様に追随している訳ではない
  • リリース地点=動作確約
    • ビルドテスト→mainにマージ→リリース
      (テストは特定のファイルの変更があったときのみ)

お一人様向け：より厳密な運用

  • reviewdogによるコードレビュー
  • 自分のコードに対して承認できない
    • github actions [bot] はworkflowが走らない
  • GitHub Appsで別人格を作成
    • agent-of-me：
      コードレビュー、変更の承認・取り消し
    • workflow-executer：
      flake.lockの更新・リリースPRの作成

外部パッケージマネージャの統合

nixは完全無欠で究極のpkg manager…
じゃない？

  • 導入にroot権限が必須
  • 多くの容量
  • 複数マシンでの/nix/storeの共有
  • 維持コスト
    • 上流の変更への追随
    • 教育・継承

= 共有鯖・支給PCで使いにくい
→nixだけで完結させるのは限界

nixos-desktop-configの構成

  • App Confはnixpkgsを優先しているものもある

folke/lazy.nvimの統合

  • lazy-loadingによる高速起動 (～80ms)
  • マルチプラットフォーム

NixOS特有の設定

  • NixOS上でのバイナリ動作
    • nix-ld+依存関係をPATHへ追加
  • lock fileをnix上で扱うロジックの構築
    • xdg.configFileはread onlyなため

(例): mason-lock.json

chezmoi

  • windowsを含めたマルチプラットフォームに対応
  • GPGやageを用いたファイル暗号化
  • 各種パスワードマネージャのサポート
  • アーカイブファイルの自動展開

強力なテンプレートとスクリプト


  • Goテンプレートによる動的な変更
  • アクティベート時のスクリプト実行による追加処理
    • 非NixOS(windows含む)でも
      宣言的パッケージインストール

chezmoiとnix

  • chezmoi: .chezmoidata.$FORMATによる状態管理
    • json、tomlはnixのbuiltinsで読み込み可
  • nix: xdg.configFile.sourceで
    設定ファイルを$XDG_CONFIG_HOME以下に配置

→設定ファイルを重複させなくて良い
(NixOS上でchezmoiは使っていない)
